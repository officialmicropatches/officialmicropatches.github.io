Launch checklist.

1 · Pages & structure

✓

Home, Shop, Custom Orders, About, FAQ, Contact

All built with mobile-first layout, consistent navigation, and footer. Active page indicated in nav.

✓

Shipping, Refund, Privacy, Terms

Starter policies written for a small US ecommerce business. Have your own counsel review before launch, or replace with Shopify-generated policies from Settings → Policies.

✓

Product detail page (PDP)

Generic template at product.html?handle=…. Pulls title, description, images, price, stock from listings.json and emits JSON-LD product schema. When SHOP_DOMAIN is set, a "Buy on Shopify" CTA appears.

✓

Internal inventory review

The earlier inventory tool is preserved at internal/inventory.html — not linked from public nav, not in sitemap, blocked in robots.txt.

2 · Shopify connection

!

SHOP_DOMAIN: micropatches.myshopify.com

Set in assets/js/shopify-config.js?v=20260520. Product page links and checkout route to the real Shopify storefront. Verify checkout end-to-end with a real test order before announcing launch.

✓

Live product/inventory sync

Storefront API token + shopify-loader.js wired up on the live site (per Claude Code handoff). Shop grid populates from Shopify, inventory is live.

○

Variant IDs for one-click cart checkout (optional polish)

Current flow: each product card opens its Shopify product page directly. Multi-item cart-bridge requires variant IDs per product. Not blocking — just polish.

✓

No fake checkout in code

Removed the "Stripe checkout opens here" placeholder. Cart drawer is hidden site-wide now that SHOP_DOMAIN is set; every Buy button opens a real Shopify product page.

3 · Custom orders

✓

Shopify Forms installed (form #992535)

Live on custom.html. Fields: Name, Email, Phone, Agency, Product Type (MicroKeychain / MicroMagnet / MicroPin / MicroCharm), Quantity, Order Details, Reference Image URL (hidden, auto-filled), How did you hear. Submissions create a Shopify customer record with all fields stored as metafields. Update the Product Type options in Shopify Forms admin from "Patch / Keychain / Coin / Other" to the four Micro* singular names if you haven't yet.

✓

Image upload working (Firebase Storage)

Patch image upload widget on custom.html writes to patch-559c8.firebasestorage.app. The Firebase download URL is auto-injected into the Shopify Forms Reference Image URL field so it lives with the customer record.

✓

Customer confirmation + admin notification

Shopify Forms sends submission notifications to OfficialMicroPatches@gmail.com and a confirmation email to the submitter. Verify both in Shopify Forms → Notifications.

○

Google Search Console verification

5 · Analytics

○

Google Analytics

Easiest path: enable GA in Shopify's Online Store → Preferences, which injects it onto Shopify-hosted pages. For this front-end, paste your GA4 measurement tag into the <head> of each page (or ask me to add it once you have the ID).

○

Microsoft Clarity, Meta Pixel

Same pattern as GA — paste the snippets when you have IDs.

6 · Domain & hosting

○

Deploy this front-end

Upload these files to your host (Cloudflare Pages, Netlify, Vercel, GitHub Pages, or your existing webhost). All static, no build step needed. The internal/ folder can be deployed too (it's just hidden from nav and indexing).

○

DNS for officialmicropatches.com

Point the apex (A/AAAA or ALIAS) and www CNAME at your host. Issue HTTPS (auto on most hosts). If you want the Shopify-hosted checkout to live on a subdomain (e.g. shop.officialmicropatches.com), set that up in Shopify admin → Settings → Domains.

7 · Email

○

Shopify customer notifications

Order confirmation, shipping confirmation, etc. are sent by Shopify automatically. Customize copy in Settings → Notifications. Verify OfficialMicroPatches@gmail.com as your sender address.

!

Custom-order submission notifications

Default mailto flow opens the customer's email app — you receive the request via email. To get a server-side notification independent of the customer's email client, install Shopify Forms (recommended) or deploy a serverless function.

8 · Manual testing before launch

○

End-to-end purchase test

Open shop → click any product → follow to Shopify → complete a real order with a real card. Confirm it appears in Shopify Orders. (Pending per Claude Code handoff — task #9.)

○

Custom order test

Fill out the custom form on the live site, upload a patch image, submit. Confirm: (1) admin email arrives at OfficialMicroPatches@gmail.com, (2) submitter receives a confirmation email, (3) Shopify customer record is created with all metafields populated, (4) Firebase image URL is present in the Reference Image URL metafield, (5) the image actually opens from that URL.

○

Shopify Forms visual styling

The embed renders with Shopify's default light theme inside a custom element / possibly shadow DOM — it may not inherit the site's dark theme. After deploy, inspect the rendered form on a real device and either override styling or accept the visual gap.

○

Mobile test (iPhone & Android)

Tap navigation, scroll the catalog, open a product, run the file upload on the custom form. Confirm no horizontal scroll, all tap targets > 44px, text readable.

○

Link audit

Click every footer link, every nav link, every CTA. Confirm no 404s.

Things this front-end build deliberately doesn't do

  • Push to Shopify Admin API directly from the browser. Admin tokens must never live in client-side code. Server-side ops (like draft-order creation) live in a serverless function or the Shopify Forms app.
  • Replace lawyer-reviewed legal policies. The drafts in this build are reasonable defaults but should be reviewed by counsel familiar with your business and jurisdiction.
  • Set up your DNS or deploy itself. The files are ready; deploy is push-button (Cloudflare Pages or GitHub Pages).
